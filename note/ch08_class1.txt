ch08_객체 지향 프로그램의 기본 코딩 방법

1. 객체지향 프로그래밍의 특징
	1-1. 캡슐화와 데이터 은닉
	     -  캡슐화(Encapsulation) :
	        객체가 포함된 속성과 메소드를 객체간 감추거나 접근 권한을 조정하는 것을 말함.
	        여기에 사용되는 keyword로 접근 제어자(access modifier)가 있다.
	1-2. 다형성 : 같은 모습으로 다른 기능들 수행 cf오버로딩
	1-3. 상속
2. 구성요소
	- 패키지명
	- 클래스명 : class명은 반드시 대문자로 시작, 알파벳, 숫자, _, $가 사용 가능하지만
		    $는 오라클사에서만, _는 되도록 사용x
	- 데이터  (인스턴스 변수 = 멤버변수, 필드): 이 데이터는 생성자나 setter를 이용해서 초기화 하지 않으면 
					      객체는 null, 숫자는 0, boolean은 false로 초기화.
	- 생성자 함수 : 리턴타입이 없는 메소드를 생성자라함. 만약 생성자가 없다면 
 		       JVM이 default 생성자를 만들어줌. (new 연산자로 호출되는 메소드)
	- 메소드
	- Getter & Setter
3. this 키워드
	3-1. 클래스의 속성과 생성자/메소드의 매개변수의 이름이 같은 경우 사용
	3-2. 클래스에 오버로딩된 다른 생성자 호출에 사용하여 코드의 중복을 피함
