ch11_추상클래스

1. 추상클래스 : 사용자가 override 할 객체를 미리 추상적으로 정의해 두는 클래스
	1-1. 문법 : abstract(추상클래스 및 추상 메소드를 선언하는 예약어)이용
		 ex) public abstract class ClassName {
	1-2. 추상클래스에는 하나 이상의 추상 메소드가 포함. 추상메소드는 정의만 하고 구현은 하지 않음
	1-3. 추상메소드에는 메소드의 선언부만 있고 실행부(구현부, Body)는 없음
	      abstract 리턴타입 methodName([매개변수])
	1-4. 추상 클래스에서는 메소드 선언만 하고 실제로 구현은 상속받는 클래스에서 함 
       	      추상클래스에서 정의된 추상적인 기능은 하위(자식) 클래스에서 상세 구현
	1-5. 클래스의 프레임만 구성. 직접 객체 생성 불가능(abstract는 인스턴스화를 금지하는 키워드)
2. 상속기능 사용중 하위 클래스에서 강제로 무엇인가 꼭 재정의 해서 사용해야 될 경우 추상클래스 개념을 이용
3. 추상클래스 상속과 일반적인 상속의 차이
	3-1 일반적 상속의 경우 부모 클래스의 메소드를 자식클래스에서 선택적으로 변경
	3-2 추상클래스를 상속한 경우 자식 클래스에서 추상 메소드 재정의는 필수
4. final (제한자)
	4-1 재정의가 필수로 이뤄져야하는 추상 클래스의 반대 개념
	4-2 클래스 앞에 붙일 경우 : 상속 금지
	     ex) public final class Test {
	4-3 멤버 메소드 앞에 붙일 경우 : 오버라이딩 금지
	     ex) public final void print(){
	4-4 멤버변수 앞에 붙일 경우 : 상수화(변경금지).
	     ex) public final int PORT_NUMBER = 80;
