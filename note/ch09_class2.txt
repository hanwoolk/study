ch9. 접근제한과 static, 패키지

1. 접근제한자
	1-1. 접근제한이란 클래스의 데이터나 메소드에 대해서 다른 클래스로부터 접근을 제한하는 것
	1-2. public : 접근을 제한하지 않는다. 다른 모든 클래스에서 사용 가능
	1-3. protected : 해당 클래스와 동일한 패키지에 있거나 상속받은 클래스일 경우에만 사용 가능
	1-4. default : 접근제어자를 명시하지 않은 경우의 디폴드 접근제어자. 같은 패키지내의 클래스들은 public 권한을 갖고 접근가능
	1-5. private : 해당 클래스만이 이 멤버를 사용할 수 있다. 외부객체에서는 접근 불가
2. static : 클래스변수(=static 변수)
	2-1. 객체변수(객체속성)는 객체가 생성될 때마다 각 객체 안의 속성 변수들이 생성되지만, 
	      클래스 변수는 클래스로부터 생성된 객체들의 수와 상관없이 하나만 생성
	2-2. 한 클래스로부터 생성된 모든 객체들은 클래스 변수를 공유
	2-3. 클래스 변수를 이용하여 객체들 사이의 통신에 사용하거나 객체들의 공통속성을 나타낼 수 있음
	2-4. 객체변수와 달리 클래스 변수(static 변수)는 클래스 이름을 통하여 접근
	2-5. 사용할 때는 new 연산자를 통해 객체로 만들어서 사용
3. static과 객체와의 관계
	3-1. 클래스 하나에서 무한대로 객체를 생성할 수 있고 그러한 객체는 모양은 동일할지 모르지만, 전혀 다른 객체
	3-2. static이 붙은 변수는 객체 변수가 아닌 클래스 변수로써 객체가 생성되기 전에 이미 존재합니다
	3-3. 객체 생성과 상관없이 사용할 수 있는 필드와 메소드에 활용할 수 있음
4. static의 장점과 단점
	4-1. 장점 : 객체 생성을 하지 않고도 사용할 수 있음
	4-2. 단점 : 가비지콜렉터의 관리 밖에 있기 때문에 항상 메모리에 상주해 있어
		  시스템이 오랜 시간 동안 돌아가게 되면 시스템 운영속도가 점차 느려짐
5. 패키지란?
	5-1. 프로그래밍에서 여러 클래스를 관리하기 위해 기능적으로 영향을 미칠 수 있는 클래스끼리 묶어 ,
	      접근 범위 안에 효과적으로 호출하기 위해서 사용하는 개념
	5-2. 윈도우의 윈도우 탐색기가 폴더 구조를 한눈에 볼 수 있도록 만들어진 프로그램인 것처럼
	      많은 .java 파일을 효율적으로 관리하기 위해 JAVA에서는 ‘패키지’라는 구조를 사용
	5-3. import의 이해 : 서로 다른 패키지의 클래스를 사용하려고 할 때는 반드시 import 키워드를 사용해서 해당 클래스를 import 해야함
